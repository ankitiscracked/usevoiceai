title: Quickstart (Client)
description: Minimal React and Vue setups to capture speech and play TTS.
---

## React

```tsx
import { useSpeech, useVoice } from "@usevoiceai/react";

export function App() {
  const { startRecording, stopRecording, status, speechStream } = useVoice({
    socketOptions: { url: import.meta.env.VITE_USEVOICEAI_WS_URL },
  });
  const { stop } = useSpeech({ speechStream });

  return (
    <div>
      <p>{status.stage === "recording" ? "Recording..." : "Idle"}</p>
      <button onClick={startRecording}>Start</button>
      <button onClick={stopRecording}>Stop</button>
      <button onClick={stop}>Stop audio</button>
    </div>
  );
}
```

Notes:

- Requires HTTPS (or `localhost`) for mic access.
- WebM/Opus capture; modern Chromium/Firefox/Safari supported.
- WebSocket URL must point to your voice session endpoint.
- If your server emits custom events, pass `onCustomEvent` to `useVoice` to handle them.

## Vue

```vue
<script setup lang="ts">
import { useVoice } from "@usevoiceai/vue";

const {
  status,
  startRecording,
  stopRecording,
  cancelRecording,
  speechStream,
} = useVoice({
  socketOptions: { url: import.meta.env.VITE_USEVOICEAI_WS_URL },
});
</script>

<template>
  <div>
    <p>{{ status.stage }}</p>
    <button @click="startRecording">Start</button>
    <button @click="stopRecording">Stop</button>
    <button @click="cancelRecording">Cancel</button>
  </div>
</template>
```

### Behavior

- Stages: `idle` → `recording` → `processing` → `completed` (or `error`).
- `speechStream` is an async iterable of PCM audio; `useSpeech` plays it via Web Audio API.
- Modes:
  - `manual` (default): you stop with `stopRecording` (or `cancelRecording`). Recording stays open until you do.
  - `auto`: if your STT provider emits `speech-end` hints, the server finalizes a turn when you stop talking; recording can restart automatically so you can keep speaking. If no hints arrive, it behaves like manual and you should call `stopRecording`.
