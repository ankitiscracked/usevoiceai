title: Quickstart (Client)
description: Minimal React and Vue setups to capture speech and play TTS.
---

## React

```tsx
import { useAudio, useVoice } from "@usevoiceai/react";

export function App() {
  const { startRecording, stopRecording, status, audioStream } = useVoice({
    socketOptions: { url: import.meta.env.VITE_USEVOICEAI_WS_URL },
  });
  const { stop } = useAudio({ audioStream });

  return (
    <div>
      <p>{status.stage === "recording" ? "Recording..." : "Idle"}</p>
      <button onClick={startRecording}>Start</button>
      <button onClick={stopRecording}>Stop</button>
      <button onClick={stop}>Stop audio</button>
    </div>
  );
}
```

Notes:

- Requires HTTPS (or `localhost`) for mic access.
- WebM/Opus capture; modern Chromium/Firefox/Safari supported.
- WebSocket URL must point to your voice session endpoint.
- If your server emits custom events, pass `onCustomEvent` to `useVoice`/`useVoiceCommand` to handle them.

## Vue

```vue
<script setup lang="ts">
import { useVoiceCommand } from "@usevoiceai/vue";

const {
  status,
  startRecording,
  stopRecording,
  cancelRecording,
  audioStream,
} = useVoiceCommand({
  socketOptions: { url: import.meta.env.VITE_USEVOICEAI_WS_URL },
});
</script>

<template>
  <div>
    <p>{{ status.stage }}</p>
    <button @click="startRecording">Start</button>
    <button @click="stopRecording">Stop</button>
    <button @click="cancelRecording">Cancel</button>
  </div>
</template>
```

### Behavior

- Stages: `idle` → `recording` → `processing` → `completed` (or `error`).
- `audioStream` is an async iterable of PCM audio; `useAudio` plays it via Web Audio API.
- Auto speech end: set `speechEndDetection: { mode: "auto" }` if your server/provider supports it.
